1. Overview

This guide covers how to run E2E (Playwright) test cases against the Dev environment (https://dev.nudgebee.pollux.in) as well as the existing Test environment. A single environment flag controls which URL, tenant, and cluster are used — no test code needs to change.

2. Prerequisites

  • Node.js LTS installed (node -v should return v18+)
  • Inside app-e2e-tests/, run npm ci to install all dependencies (includes cross-env and Playwright)
  • Download Chromium browser: 
    npx playwright install chromium
  • Confirm .env and .env.dev files exist in app-e2e-tests/
  • Confirm e2e-config.json exists in app-e2e-tests/

6. Running Test Cases

Working directory All commands below must be run from inside the app-e2e-tests/ folder. 
cd app-e2e-tests

6.1 Shortcut Script – Dev Only (Recommended)

Logic lives in dt.js (cross-platform). Platform-specific wrappers delegate to it — same flags work everywhere.

PlatformCommandFirst-time setup
Windows.\dt <args>None
Mac / Linux./dt.sh <args>chmod +x dt.sh (once)
Any (universal)node dt.js <args>None
Run specific file – Dev
# Windows
.\dt tests/ClusterDetails/Monitoring/MonitoringAlertManager.spec.ts
# Mac/Linux
./dt.sh tests/ClusterDetails/Monitoring/MonitoringAlertManager.spec.ts
Run entire folder – Dev (one by one)
# Windows
.\dt tests/ClusterDetails/ --workers=1
# Mac/Linux
./dt.sh tests/ClusterDetails/ --workers=1
Run Cloud Account – AWS
# Windows
.\dt tests/CloudAccount/AWS --workers=1
# Mac/Linux
./dt.sh tests/CloudAccount/AWS --workers=1
Run by test name – Dev
# Windows
.\dt --grep "API testing Cluster Details"
# Mac/Linux
./dt.sh --grep "API testing Cluster Details"

6.2 Full Commands – Dev Environment

# Specific test file
npx cross-env E2E_ENVIRONMENT=dev npx playwright test --project=chromium tests/ClusterDetails/Monitoring/MonitoringAlertManager.spec.ts --headed

# Specific folder, sequential
npx cross-env E2E_ENVIRONMENT=dev npx playwright test --project=chromium tests/ClusterDetails/ --headed --workers=1

# By test name (grep)
npx cross-env E2E_ENVIRONMENT=dev npx playwright test --project=chromium --grep "API testing Cluster Details" --headed

# Full suite
npx cross-env E2E_ENVIRONMENT=dev npx playwright test --project=chromium --headed

Required GitHub Secret

The dev workflow uses the same secret as the test workflow — no new secret needed.

Secret NameContent
E2E_TEST_ENVFull contents of .env file + the 4 DEV_* vars below

Add these 4 lines to the E2E_TEST_ENV secret in GitHub:

DEV_BASE_URL=https://dev.nudgebee.pollux.in
DEV_SWITCH_TENANT=Nudgebee
DEV_CLUSTER_NAME=k8s-dev
DEV_SLACK_WEBHOOK_URL=https://hooks.slack.com/services/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
How it works in CI The workflow writes .env from the secret, then extracts all DEV_* lines and strips the prefix to create .env.dev. Playwright loads .env.dev first (dev values win), then cascades to .env for integration credentials.

Add Secret in GitHub

  1. Go to: GitHub repo → Settings → Secrets and variables → Actions
  2. Click E2E_TEST_ENV → Edit
  3. Append the 4 DEV_* lines shown above at the bottom
  4. Click Update secret
