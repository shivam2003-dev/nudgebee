# Security Policy

## Reporting a vulnerability

Please **do not** file public GitHub issues for security vulnerabilities — they're visible to anyone watching the repo before a fix exists.

Use [GitHub's private vulnerability reporting](../../security/advisories/new) instead. Reports stay between you and the maintainers until coordinated disclosure.

We aim to acknowledge reports within 3 business days and provide a status update within 7 business days.

## Supported versions

Security fixes target the latest commit on `main`. Older releases are not back-ported unless specifically requested.
